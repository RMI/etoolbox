etoolbox.datazip.core.obj_from_recipe(thing, module, klass=None, constructor=None)[source]#

Recreate an object from a recipe.

Can be module.klass(thing), module.constructor(thing), or module.klass.constructor(thing).

Parameters:

  • thing (dict | tuple | list | str | int | float | BytesIO) – what will be passed to constructor / __init__

  • module (str) – the module that the function or class can be found in

  • klass (str | None) – The __qualname__ of the class, if needed. If None, then constructor must be a function.

  • constructor (str | None) – A constructor method on a class if klass is specified, otherwise a function that returns the desired object. If None, the class klass will be created using its __init__.

Return type:

Any

class etoolbox.datazip.core.DataZip(file, mode='r', recipes=None, *args, **kwargs)[source]#

Bases: zipfile.ZipFile

SubClass of ZipFile with methods for easier use with utils.

z = DataZip(file, mode=”r”, compression=ZIP_STORED, allowZip64=True,

compresslevel=None)

Open the ZIP file.

Parameters:

  • file (str | Path | BytesIO) – Either the path to the file, or a file-like object. If it is a path, the file will be opened and closed by ZipFile.

  • mode – The mode can be either read ‘r’, write ‘w’, exclusive create ‘x’, or append ‘a’.

  • recipes – add more customization on how attributes will be stored organized like etoolbox.datazip.core.RECIPES

  • compression – ZIP_STORED (no compression), ZIP_DEFLATED (requires zlib), ZIP_BZIP2 (requires bz2) or ZIP_LZMA (requires lzma).

  • allowZip64 – if True ZipFile will create files with ZIP64 extensions when needed, otherwise it will raise an exception when this would be necessary.

  • compresslevel – None (default for the given compression type) or an integer specifying the level to pass to the compressor. When using ZIP_STORED or ZIP_LZMA this keyword has no effect. When using ZIP_DEFLATED integers 0 through 9 are accepted. When using ZIP_BZIP2 integers 1 through 9 are accepted.
